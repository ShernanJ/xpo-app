# Anchor Summary Exemplars

Use these as the format for summarizing scraped posts.
Goal: provide style + topics + proof patterns without copying text.

---

## Example A — Builder (casual), 0→1k, product in public

**Recurring topics**
- shipping updates (small increments)
- “what broke / what fixed it”
- building an agent / chrome extension / scraping

**Voice + cadence**
- mostly lowercase
- short lines, frequent line breaks
- low emoji, direct language
- uses contrast: “i thought X… it was Y”

**Best-performing patterns**
- hook: a surprising bug or realization
- includes one concrete detail (stack choice, file path, metric)
- ends with a soft question or “want the template?”

**Avoid**
- inspirational “aha moment” framing
- generic “tips” list intros
- corporate words

---

## Example B — Builder (pro), 1k–10k, technical authority

**Recurring topics**
- architecture decisions
- tradeoffs and reliability
- structured frameworks

**Voice + cadence**
- crisp sentences
- 1 hook + 3 bullets
- minimal slang, minimal emojis

**Best-performing patterns**
- clear claim with “why”
- provides a replicable checklist
- references real constraints and outcomes

**Avoid**
- motivation-first storytelling
- overlong threads without evidence
